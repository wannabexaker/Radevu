# Verticals — Locked profession list for Radevu

Seven professional verticals Radevu supports. Phase 1 stays **generic** (any business can use the same Service model with custom names). Phase 2 adds **per-profession templates** (pre-built service sets + one-tap action menus) for these seven specifically.

This doc serves three purposes:

1. **Seed data + demo content** — when populating dev/test, draw from these examples
2. **Marketing copy reference** — when writing landing features, use these professions as concrete examples (avoid abstract "professionals")
3. **Phase 2 template scope** — pre-built service sets only for these seven, not arbitrary verticals

Prices are **suggestions only**. Every business sets their own. Durations are typical defaults.

## Phase 1 demo seed business

For dev/test/screenshots, the seeded `test-shop` is a **κουρέας**. Universal, instantly relatable, services are short (good for quick-booking demos). When chunks need richer demo data, add a second seed: **οδοντίατρος** as `test-dentist` (longer durations + follow-up templates demonstrate the CRM angle).

## Vocabulary rules for these verticals

Every email, push, and dashboard string that mentions an appointment for any of these verticals must follow `docs/DESIGN.md` §10 "ραντεβού is ambiguous" rules — always anchored with service or business name. The cafe HR vertical is especially sensitive: "ραντεβού με τον πελάτη" must always be "συνέντευξη", never just "ραντεβού" alone.
